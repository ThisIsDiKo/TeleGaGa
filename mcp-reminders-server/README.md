# MCP Reminders Server

MCP сервер для управления напоминаниями в TeleGaGa боте.

## Инструменты

### 1. create_reminder
Создание нового напоминания.

**Параметры:**
- `chatId` (string) - ID чата пользователя
- `text` (string) - Текст напоминания
- `dueDate` (string) - Дата напоминания в формате YYYY-MM-DD

**Пример:**
```json
{
  "chatId": "702588940",
  "text": "Купить молоко",
  "dueDate": "2026-01-29"
}
```

### 2. get_reminders
Получение напоминаний за период.

**Параметры:**
- `chatId` (string) - ID чата пользователя
- `dateFrom` (string, опционально) - Начальная дата в формате YYYY-MM-DD
- `dateTo` (string, опционально) - Конечная дата в формате YYYY-MM-DD

**Пример:**
```json
{
  "chatId": "702588940",
  "dateFrom": "2026-01-28",
  "dateTo": "2026-01-31"
}
```

### 3. delete_reminder
Удаление/завершение напоминания.

**Параметры:**
- `chatId` (string) - ID чата пользователя
- `reminderId` (string) - UUID напоминания

**Пример:**
```json
{
  "chatId": "702588940",
  "reminderId": "uuid-1234-5678"
}
```

## Установка

```bash
npm install
```

## Запуск

```bash
node index.js
```

## Хранилище

Напоминания хранятся в `storage/<chatId>.json`.

Каждый файл имеет структуру:
```json
{
  "chatId": "702588940",
  "reminders": [
    {
      "id": "uuid-1234-5678",
      "text": "Купить молоко",
      "dueDate": "2026-01-28",
      "createdAt": "2026-01-28T14:30:00+03:00",
      "completed": false
    }
  ]
}
```

## Безопасность

- Каждый chatId работает только со своим файлом
- Валидация всех параметров перед обработкой
- Изоляция файловой системы (нет доступа за пределы storage/)
